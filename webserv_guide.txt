1) Start from process lifecycle (entrypoint)
Read main in main.cpp, then jump to Config::Config and Config::run.

Why first: this gives the full control flow: boot → parse config → open sockets → event loop.

Key symbols:

main
Config::Config
Config::run
ft_sigint_handler
2) Understand configuration model before networking
Read how c.conf becomes objects:

Config::_parseServer
Config::_parseLocation
Config::_parseListen
Location::setRootPath, Location::setAllowMethods, Location::addCgi
Server::addLocation, ServersSocket::addServer
Why next: all runtime behavior depends on these objects.

3) Then read socket/event-loop wiring
Focus on:

Config::_initSockets
Config::_addPollfd, Config::_rmPollfd
Config::_addClient
Config::_pruneSocket
Why: this explains how server sockets and client sockets are mixed in poll().

4) Follow one request end-to-end
Path:

Config::_readRequest
Client::createRequest / Request::appendStr
Request::handleRequest
Request::setLocation
Config::_sendResponse
Client::createResponse
Response::Response
Why: this is the core pipeline.

5) Deep dive parser only after pipeline is clear
In Request.cpp, read in this order:

isCompletedRequest
updateHeader → getHeaderVariables
updateBody → treatingBody
multipart/urlencoded handlers: FormTwo, FormOne
Why: parser is the densest part; only do this after seeing where its output is used.

6) Study response decision tree
Read Response::Response as a chain:

Response::_checkLocation
Response::_checkResource
Response::_checkCgi
Response::_checkMethod
Response::_errorCheck
Then method handlers:

Response::_handleGet
Response::_handlePost
Response::_handleDelete
Why: this is where HTTP semantics are encoded.

7) Read CGI last
Cgi::Cgi
Cgi::_initEnv
Cgi::execute
call site: Response::_runCgi
Why: CGI adds fork/exec/pipe, so it is easier once request/response is already clear.

8) Practical progression plan (for your own webserver)
Implement minimal GET static file.
Add config parser for listen/root.
Add poll loop + accept/read/write.
Add routing (location longest prefix like Request::setLocation).
Add status/error pages.
Add POST upload.
Add DELETE.
Add CGI.
